# 와인 x 콜키지 매칭 서비스 MVP PRD

## 1. 핵심 정보

**목적**: 콜키지 식당을 방문할 때 가져갈 와인에 잘 어울리는 메뉴를 AI 소믈리에가 추천해준다.

**사용자**: 와인을 즐기지만 메뉴 페어링에 자신이 없는 20~40대 직장인, 콜키지 문화에 관심 있는 소비자.

---

## 2. 사용자 여정

핵심 흐름: **와인 취향 설정 → 근처 콜키지 식당 탐색 → 와인/상황 선택 → 메뉴별 페어링 점수 확인 → 메뉴 선택 → AI 소믈리에 설명**

```
1. 앱 최초 실행 (랜딩 페이지)
   └─ 회원가입 클릭
       ↓

2. 회원가입 페이지
   └─ 가입 완료 → 온보딩 페이지로 자동 이동
       ↓

3. 온보딩 페이지 (취향 설정)
   └─ 와인 취향 슬라이더 + 못먹는 음식 선택 → 저장
       ↓

4. 홈 페이지 (주변 콜키지 식당 목록)
   └─ GPS 기반 식당 목록 표시 (메뉴 데이터 있는 식당만 노출)
       ├─ 필터/정렬 적용 → 목록 갱신
       └─ 식당 카드 클릭
           ↓

5. 식당 상세 페이지
   ├─ 와인 선택 (내 와인 컬렉션 또는 취향 프로파일 기본값 사용)
   ├─ 상황 선택 태그 (데이트 / 친구 모임 / 비즈니스 / 혼술 / 가족 모임)
   └─ 메뉴 목록 + 페어링 점수 표시 → 메뉴 선택
       ↓

6. AI 매칭 페이지 (메뉴 선택 후)
   └─ 선택한 메뉴와 와인의 페어링 이유 + 상황 맞춤 소믈리에 설명 표시
       └─ 방문 후 피드백 제출 (좋아요/별로 + 자유 텍스트 후기) → 취향 벡터 자동 업데이트

7. 마이페이지
   ├─ 취향 프로파일 수정
   ├─ 취향 변화 시각화 (레이더 차트 시계열)
   └─ 내 와인 컬렉션 (와인 목록 조회/추가/삭제)
```

## 3. 기능 명세

### 3.1 핵심 기능


| ID       | 기능명             | 설명                                                       | 관련 페이지        |
| -------- | --------------- | -------------------------------------------------------- | ------------- |
| **F-01** | 취향 온보딩          | 당도/산미/바디감/탄닌 슬라이더 + 선호 향 + 못먹는 음식 선택                     | 온보딩           |
| **F-02** | 취향 슬라이더 UI      | 4개 슬라이더 + 레이더 차트 실시간 시각화                                 | 온보딩, 마이페이지    |
| **F-03** | 취향 수정           | 마이페이지에서 취향 프로파일 언제든지 수정                                  | 마이페이지         |
| **F-04** | 콜키지 식당 검색       | 사용자가 선정한 지역의 GPS 기반 반경 2km 내 콜키지 허용 식당 목록 (메뉴 있는 식당만 표시) | 홈             |
| **F-05** | 식당 상세 조회        | 메뉴 목록 + 페어링 점수 표시, 콜키지 조건/운영시간 표시                        | 식당 상세         |
| **F-06** | 필터 및 정렬         | 음식 카테고리/콜키지 무료 여부/거리순/별점순                                | 홈             |
| **F-11** | 메뉴 맛 프로파일 벡터화   | 메뉴별 6차원 맛 벡터 생성 후 pgvector 저장 (운영자 수기 입력 후 백그라운드)        | 내부 처리         |
| **F-12** | 메뉴 페어링 점수 표시    | 식당 상세에서 각 메뉴에 와인 특성 기준 매칭 점수 아이콘 표시                      | 식당 상세         |
| **F-13** | AI 페어링 설명 생성    | 선택한 메뉴, 와인, 상황을 반영한 소믈리에 스타일 설명 2~3문장                    | AI 매칭         |
| **F-14** | 취향 진화 트래킹       | 피드백 + 텍스트 후기 기반 취향 벡터 업데이트 + 시계열 레이더 차트 시각화              | 마이페이지, AI 매칭  |
| **F-15** | 와인 컬렉션 관리       | 마이페이지에서 내 와인 목록 조회/추가/삭제                                 | 마이페이지         |
| **F-16** | 와인 라벨 OCR 등록    | 라벨 사진 촬영 → OCR → GPT 파싱 → wines 테이블 저장                   | 마이페이지 (와인 추가) |
| **F-17** | 상황 컨텍스트 선택      | 식당 상세에서 오늘의 상황 태그 선택 (데이트/친구 모임/비즈니스/혼술/가족 모임)           | 식당 상세         |
| **F-18** | 텍스트 후기 기반 취향 학습 | 후기 텍스트 → GPT 분석 → 취향 프로파일 수치 정밀 업데이트                     | AI 매칭         |

## 4. 메뉴 구조

```
[비로그인 상태]
├─ 랜딩 페이지
├─ 로그인
└─ 회원가입

[로그인 후 공통 하단 탭]
├─ 홈 (주변 식당 목록)
├─ AI 매칭 결과
└─ 마이페이지

[홈 → 식당 상세]
└─ 메뉴 데이터 있는 식당만 표시
    └─ 와인 선택 + 상황 선택 → 메뉴 목록 (페어링 점수) → 메뉴 선택 → AI 매칭

[AI 매칭 페이지 내]
└─ 방문 후 피드백 (좋아요/별로 + 텍스트 후기) → 취향 업데이트

[마이페이지 내]
├─ 취향 수정
├─ 취향 변화 시각화
└─ 내 와인 컬렉션 (와인 목록 / 와인 추가 / 라벨 OCR 등록)
```

## 8. 식당 데이터 전략

네이버 Local Search API는 식당 기본 정보(이름, 주소, 좌표, 전화번호, 카테고리)만 제공한다. 콜키지 조건과 메뉴 데이터는 API로 수집할 수 없으므로 아래 전략을 사용한다.

### 네이버 API 제공 범위


| 데이터             | 제공 여부           |
| --------------- | --------------- |
| 식당명, 주소, 좌표     | 제공              |
| 전화번호, 음식 카테고리   | 제공              |
| 콜키지 허용 여부       | 미제공             |
| 콜키지 비용 / 병 수 제한 | 미제공             |
| 메뉴 항목 / 가격      | 미제공             |
| 영업시간            | 불안정 (미제공 케이스 多) |


### 초기 데이터 구축 (운영자 직접 입력)

- 강남/홍대 콜키지 식당 50곳을 네이버지도로 직접 조사
- 콜키지 허용 여부, 비용, 병 수 제한, 예약 필요 여부 확인 후 Supabase에 수기 입력
- 식당당 대표 메뉴 5~10개 직접 입력 (별도 admin 페이지 없이 Supabase 대시보드 직접 사용)
- 메뉴 입력 후 맛 프로파일 벡터화는 백그라운드 스크립트로 처리

### 홈 노출 기준

- menus 테이블에 해당 restaurant_id로 1개 이상의 메뉴가 존재하는 식당만 홈에 표시
- 메뉴 미입력 식당은 DB에 보관하되 유저에게 노출하지 않음

### 별점 및 콜키지 조건 관리

- 네이버 별점을 초기값으로 사용, 피드백 누적에 따라 앱 내 별점 진화 (베이지안 가중 평균)
  - 공식: `rating = (naver_rating × 10 + Σ feedback_score) / (10 + feedback_count)`
  - 좋아요 = 5점, 별로 = 1점
- 운영자 수기 입력 시 네이버 플레이스에서 확인한 별점을 `naver_rating`으로 함께 입력
- 사용자가 콜키지 조건 변경 신고 → 운영자 확인 후 반영
