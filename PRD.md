# 와인 x 콜키지 매칭 서비스 MVP PRD

## 1. 핵심 정보

**목적**: 콜키지 식당 방문 시 AI 소믈리에가 두 가지 방향으로 페어링을 추천한다.
- ① **메뉴 → 와인 추천**: 식당 메뉴를 선택하면 어울리는 와인 추천 (내 컬렉션 우선, 없으면 전체 DB)
- ② **내 와인 → 메뉴 추천**: 내가 가진 와인을 기준으로 어울리는 식당/메뉴 추천

**앱 형태**: 모바일 웹앱 (max-w-[390px], Kotlin 네이티브 전환 예정)

**사용자**: 와인을 즐기지만 메뉴 페어링에 자신이 없는 20~40대 직장인, 콜키지 문화에 관심 있는 소비자.

---

## 2. 사용자 여정

핵심 흐름: **와인 취향 설정 → 근처 콜키지 식당 탐색 → 와인/상황 선택 → 메뉴별 페어링 점수 확인 → 메뉴 선택 → AI 소믈리에 설명**

```
1. 앱 최초 실행 (랜딩 페이지)
   └─ 회원가입 클릭
       ↓

2. 회원가입 페이지
   └─ 가입 완료 → 온보딩 페이지로 자동 이동
       ↓

3. 온보딩 페이지 (취향 설정)
   └─ 와인 취향 슬라이더 + 못먹는 음식 선택 → 저장
       ↓

4. 홈 페이지 (취향 성운 + 맞춤 추천)
   ├─ 취향 성운(星雲) 시각화 — 탄닌/산미/바디/당도 Canvas 파티클 (크게 배치)
   ├─ 내 취향 맞춤 와인 추천 가로 스크롤 목록
   └─ 근처 콜키지 식당 카드
       ├─ [메뉴 → 와인] 탭 선택 → 식당 카드 클릭
       └─ [내 와인 → 메뉴] 탭 선택 → 마이페이지 컬렉션으로 이동
           ↓

5. 식당 상세 페이지
   └─ 메뉴 목록 표시 → 메뉴 선택
       → AI 와인 추천: 내 컬렉션에서 최적 와인 우선 → 없으면 전체 와인 DB 추천
           ↓

6. AI 매칭 페이지 (메뉴 선택 후)
   └─ 추천 와인 + 페어링 이유 소믈리에 설명 표시 (내 컬렉션 우선 강조)
       └─ 방문 후 피드백 제출 (좋아요/별로 + 자유 텍스트 후기) → 취향 벡터 자동 업데이트

7. 마이페이지
   ├─ 취향 프로파일 수정
   ├─ 취향 변화 시각화 (레이더 차트 시계열)
   └─ 내 와인 컬렉션 (와인 목록 조회/추가/삭제)
```

## 4. 메뉴 구조

```
[비로그인 상태]
├─ 랜딩 페이지
├─ 로그인
└─ 회원가입

[로그인 후 공통 하단 탭 — 3개]
├─ 홈     (취향 성운 + 맞춤 와인 추천 + 근처 식당)
├─ 지도   (콜키지 식당 탐색/검색)
└─ 마이페이지

[홈 — 두 가지 흐름 진입]
├─ [메뉴 → 와인] 탭: 근처 식당 → 식당 상세 → 메뉴 선택 → AI 와인 추천 (내 컬렉션 우선)
└─ [내 와인 → 메뉴] 탭: 마이페이지 컬렉션 → 와인 선택 → 어울리는 식당/메뉴 AI 추천

[식당 상세]
└─ 메뉴 목록 → 메뉴 선택 → AI 와인 추천 → AI 매칭 페이지

[AI 매칭 페이지 내]
└─ 방문 후 피드백 (좋아요/별로 + 텍스트 후기) → 취향 업데이트 → 홈 성운 반영

[마이페이지 내]
├─ 취향 성운 시각화 + 수치 바
├─ 취향 재설정 (온보딩)
└─ 내 와인 컬렉션 (와인 목록 / 와인 추가 / OCR 탭 /ocr)
```

## 8. 식당 데이터 전략

네이버 Local Search API는 식당 기본 정보(이름, 주소, 좌표, 전화번호, 카테고리)만 제공한다. 콜키지 조건과 메뉴 데이터는 API로 수집할 수 없으므로 아래 전략을 사용한다.

### 네이버 API 제공 범위


| 데이터             | 제공 여부           |
| --------------- | --------------- |
| 식당명, 주소, 좌표     | 제공              |
| 전화번호, 음식 카테고리   | 제공              |
| 콜키지 허용 여부       | 미제공             |
| 콜키지 비용 / 병 수 제한 | 미제공             |
| 메뉴 항목 / 가격      | 미제공             |
| 영업시간            | 불안정 (미제공 케이스 多) |


### 초기 데이터 구축 (운영자 직접 입력)

- 강남/홍대 콜키지 식당 50곳을 네이버지도로 직접 조사
- 콜키지 허용 여부, 비용, 병 수 제한, 예약 필요 여부 확인 후 Supabase에 수기 입력
- 식당당 대표 메뉴 5~10개 직접 입력 (별도 admin 페이지 없이 Supabase 대시보드 직접 사용)
- 메뉴 입력 후 맛 프로파일 벡터화는 백그라운드 스크립트로 처리

### 홈 노출 기준

- menus 테이블에 해당 restaurant_id로 1개 이상의 메뉴가 존재하는 식당만 홈에 표시
- 메뉴 미입력 식당은 DB에 보관하되 유저에게 노출하지 않음

### 별점 및 콜키지 조건 관리

- 네이버 별점을 초기값으로 사용, 피드백 누적에 따라 앱 내 별점 진화 (베이지안 가중 평균)
  - 공식: `rating = (naver_rating × 10 + Σ feedback_score) / (10 + feedback_count)`
  - 좋아요 = 5점, 별로 = 1점
- 운영자 수기 입력 시 네이버 플레이스에서 확인한 별점을 `naver_rating`으로 함께 입력
- 사용자가 콜키지 조건 변경 신고 → 운영자 확인 후 반영

## 9. 검증 시나리오

- **[완료] 와인 라벨 OCR GPT Vision 연동**: 실제 라벨 사진 업로드 → 이름/품종/빈티지 정상 인식 확인
- **[완료] GPT 추정값 확인**: tannin/acidity/body/sweetness 추정값이 0~5 범위 내에서 반환되는지, 유명 와인(예: Tignanello)은 합리적인 값인지 확인
- **[미완] wines DB 매칭 검증**: wines 테이블 구축 후 라벨 인식 이름과 DB 매칭 → DB 값 우선 사용 확인
- 식당 상세에서 와인/상황 선택 후 페어링 점수 변화 확인
- 와인 미선택 시 취향 프로파일 폴백 동작 확인
- 텍스트 후기 입력 → GPT 분석 → taste_profiles 수치 변경 확인
- 데이트 vs 친구모임 상황에 따른 GPT 출력 차이 확인
- 홈에서 메뉴 없는 식당 미노출 확인
